성능 및 품질

프로젝트 품질 요구사항

1 성능 요구사항#

항목 목표수치
[웹/모바일] 동시접속자 50 명이상
[백엔드] 트랜잭션 처리속도 3000 TPS이상

2 인터페이스 요구사항#

2.1 편의성#

  • 사용자에게 필요한 기능과 정보를 제공
  • 사용자의 목적의 시간과 단계를 최소화
  • 직관적이고 일관성 있게 구현
  • 사용자가 실수하지 않게 구현

2.2 일관성#

  • 정보 전달에 용이한 일관성 있는 화면 구조 제공
  • 사용자 친화적인 어휘와 일관된 메시지 사용
  • 강조/에러를 표현하는 컬러 등 의미있는 컬러를 일관되게 사용
  • 통일된 서체와 체계화된 크기의 서체 사용

2.3 접근성#

  • 모든 사용자가 이용할 수 있도록 구현
  • 대체 텍스트 제공
  • 멀티미디어 자막 제공
  • 색에 무관한 컨텐츠 제공
  • 명확한 액션을 지시
  • 텍스트 컨텐츠 명도 대비 제공

3 보안 요구사항#

3.1 보안 기능 요구사항#

  • 고객 유형에 따른 기능 차별화
  • 고객 유형에 따른 역할 인증
  • 고객에 대한 개인정보보호 규정 준수
    • 개인정보보호법, 표준개인정보보안지침 등에 따른 지침
    • 개인정보보호지침 준수

3.2 보안 취약점 요구사항#

  • OWASP Top10 에 해당하는 보안 요구사항 점검
  • KISA secure coding 가이드라인 보안 요구사항 점검

4 테스트 요구사항#

  • 통합테스트 및 인수테스트 수행

4.1 통합테스트 수립 방안#

4.1.1 단위 테스트#

  • 프로그램 작성시 표준과 절차 준수 여부
  • 상세설계사양 충족여부
  • 기능이 정확히 수행되는지 여부

4.1.2 통합테스트#

  • 연계테스트 정상 수행 여부
  • 예외상황에서 정상 수행 여부
  • 접근권한 등 보안성 충족 여부

4.1.3 인수테스트#

  • 품질보증 계획의 목표 만족 여부
  • 각 단계 별 테스트 오류 결과 반영 여부
  • 기능 요구사항 만족 여부

4.1.4 성능/시스템테스트#

  • 응용 어플리케이션의 정상동작 여부
  • 장애에 대해 업무의 연속성의 보장여부
  • 업무처리 성능기준에의 부합여부
